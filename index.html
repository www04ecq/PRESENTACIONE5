Caracteristicas

Principales Caracteristicas de una Red MPLS

Multiprotocol Label Switching (MPLS) es la tecnología estándar de la industria para redes de transporte de alta velocidad, operando en la "Capa 2.5" del modelo OSI para optimizar el reenvío de paquetes.

⚡ Velocidad de Conmutación

Los routers en el núcleo de la red (Core) no necesitan analizar las cabeceras IP complejas. Toman decisiones de reenvío basándose únicamente en una etiqueta simple de 32 bits, realizando la conmutación en hardware (CEF).

🛤️ Ingeniería de Tráfico

Permite a los administradores dirigir el tráfico por rutas óptimas que no necesariamente serían elegidas por los protocolos de enrutamiento IP tradicionales (OSPF/EIGRP), equilibrando la carga de la red.

🔒 Soporte de VPNs

Es la base para servicios como MPLS VPN (L3VPN) y VPLS (L2VPN), permitiendo aislar el tráfico de múltiples clientes de forma segura sobre la misma infraestructura física del proveedor.

Arquitectura

Elementos de la Red MPLS

🌐 LER (Label Edge Router)

Also known as PE (Provider Edge).

Son los routers de borde. Se encargan de conectar la red del cliente con la red del proveedor.

  • Ingress PE: Recibe paquetes IP, consulta la tabla FIB y inserta la etiqueta MPLS.
  • Egress PE: Recibe paquetes MPLS, elimina la etiqueta y entrega el paquete IP al cliente.

🔄 LSR (Label Switching Router)

Also known as P (Provider).

Son los routers del núcleo de la red. No conocen las redes de los clientes, su única función es recibir paquetes etiquetados, consultar su tabla LFIB, intercambiar la etiqueta y reenviar el paquete al siguiente salto.

🛤️ LSP (Label Switched Path)

El "Túnel" Virtual.

Es la secuencia de routers (desde el Ingress hasta el Egress) por donde viaja un flujo de datos específico. Se establece dinámicamente gracias al protocolo LDP y las rutas proporcionadas por OSPF.

PUSH

Se realiza en el Ingress LER. Cuando un paquete IP ingresa a la red MPLS, el router examina su tabla de enrutamiento IP (FIB), determina el FEC y le añade la cabecera MPLS con la etiqueta correspondiente al siguiente salto.

SWAP

Se realiza en los LSR (Core). El router recibe un paquete con una etiqueta de entrada, busca en su tabla LFIB qué etiqueta le corresponde al siguiente salto, retira la etiqueta vieja e inserta la nueva.

POP

Se realiza en el Penúltimo Salto (PHP). Para optimizar el router Egress, el router Core justo antes de destino retira la etiqueta MPLS y envía el paquete IP puro, ahorrando una doble búsqueda al router de salida.

Configuración Paso a Paso

La implementación de MPLS requiere una base de enrutamiento IP sólida antes de activar las etiquetas.

1️⃣ Enrutamiento Base (OSPF)

MPLS requiere que los routers conozcan la topología de red. OSPF propaga las rutas, incluyendo las interfaces Loopback vitales para LDP.

! Configuración en todos los routers (Ejemplo R3 Core)
router ospf 1
network 10.0.0.0 0.0.0.255 area 0
network 10.0.1.0 0.0.0.255 area 0
network 3.3.3.3 0.0.0.0 area 0 ! Loopback es obligatoria

2️⃣ Activación de CEF y MPLS

Cisco Express Forwarding (CEF) es un requisito obligatorio porque MPLS depende de las estructuras de datos de CEF (FIB y Adjacency Table) para el reenvío.

! Activación global
ip cef
mpls label protocol ldp

! Activación en interfaces del Core (R3 y R4)
interface FastEthernet 0/0
mpls ip

! En Routers de Borde (R1/R2), solo en interfaces hacia el Core
interface FastEthernet 1/0 ! Hacia R3/R4
mpls ip
Funcionamiento Interno

Plano de Control vs. Plano de Datos

La magia de MPLS radica en la separación estricta entre quién construye las rutas y quién reenvía el tráfico.

🧠 Plano de Control (El Cerebro)

Función: Intercambiar información de enrutamiento y mapear rutas a etiquetas. No reenvía tráfico de usuario.

  • OSPF/IS-IS: Conoce las rutas IP y construye la tabla RIB.
  • LDP (Label Distribution Protocol): Toma las rutas del OSPF y les asigna etiquetas locales, creando la tabla LIB (Label Information Base).
  • Establece la vecindad con el comando show mpls ldp neighbor.

⚡ Plano de Datos (Los Músculos)

Función: Reenviar paquetes a velocidades de alambre (Wire-speed) sin procesar IP.

  • CEF (FIB): Reenvía tráfico IP puro (entrante al Ingress).
  • LFIB (Label Forwarding Information Base): Reenvía tráfico MPLS. Consulta la etiqueta de entrada, determina la etiqueta de salida y la interfaz (PUSH/SWAP/POP).
  • Verificado con show mpls forwarding-table.
